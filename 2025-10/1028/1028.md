# 📝 알고리즘 문제 풀이 - 2025-10-28

## 문제 📖

- **문제 링크:** [백준 - 게임](https://www.acmicpc.net/problem/1072)

- **문제 유형:** 이분 탐색

- **난이도:** 실버3

- **풀이 시간:** ⏳ 1시간반

## ✍ 접근 방식

실버3인데 너무 어렵다 ㅎㅎ;
아래와 같은 트러블 슈팅이 있었다.
근데 읽어봐도 잘 모르겠다..
나중에 부동소수점에 대해서 더 공부해봐야할듯하다.

---

두 방식의 계산 순서가 달라 부동소수점 오차가 결과에 영향을 줄 수 있습니다.

**이전 코드 (사용자):**

```javascript
const Z = Math.floor((Y / X) * 100); // Y/X를 먼저 계산
const newZ = Math.floor(((Y + mid) / (X + mid)) * 100);
```

**정답 코드:**

```javascript
let z = Math.floor((100 * Y) / X); // 100*Y를 먼저 계산
let new_z = Math.floor((100 * (Y + mid)) / (X + mid));
```

**실제 차이:**

- `(Y / X) * 100`: 소수 나눗셈 후 곱셈
- `(100 * Y) / X`: 정수 곱셈 후 나눗셈

예: `(470000000 / 1000000000) * 100`과 `(100 * 470000000) / 1000000000`이 부동소수점에서 다른 중간값을 만들 수 있어, 일부 테스트 케이스에서 `Math.floor()` 결과가 달라질 수 있습니다.

백준 등에서는 두 번째 방식(`(100 * Y) / X`)이 더 안정적인 결과를 낸다고 알려져 있습니다.
